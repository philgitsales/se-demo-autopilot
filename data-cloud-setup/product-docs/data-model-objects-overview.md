DATA 360
Data Model Objects (DMOs)

Create, view, and manage data model objects (DMOs). DMOs are fundamental to Data 360. Familiarize yourself with the definitions, concepts, and resources related to DMOs before creating or working with them.

View Your Data Model Objects
You can view your Data 360 data model objects in either list or graph view.
Create a Custom Data Model Object
If a standard data model object doesn’t meet a business need, you can create a custom data model object (DMO) to store information that is unique to your business. You can create a custom data model object from an existing DMO, from a file, or from scratch.
Create a Custom Data Model Object from an Existing Data Model Object
Use a custom data model object (DMO) to store information that is unique to your business. You can create a DMO by copying an existing standard or custom DMO.
Create a Custom Data Model Object from a CSV File
Use a custom data model object (DMO) to store information that is unique to your business. You can add a custom DMO by uploading a CSV file.
Edit Data Model Object Properties
You can edit Data Model Object (DMO) properties either on the Data Model tab or on the data mapping page.
Change the Category of a Data Model Object
In most cases, a data model object’s category can’t be changed after the object is created. The category can be changed only if it’s changing from Profile to Other or from Other to Profile on an object that isn’t used in segmentation or activation.
Data Model Object Relationships
A Data 360 data model object (DMO) can have a standard or custom relationship with other DMOs. The relationship can be structured as a one-to-one or many-to-one relationship. You can view the relationships and their statuses on a DMO record’s Relationships tab.
